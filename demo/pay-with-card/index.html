Invoice

Pay with Card

Create an Acumatica payment, apply the configured card fee, and process it through linked2pay.

Start from a released invoice

Open a released invoice with a remaining balance and click Pay. Select the customer's active linked2pay card profile and its approved cash account.

Review the automatic processing fee

When an eligible linked2pay card payment is saved, the configured percentage is applied automatically. The integration creates and applies a CCFEE debit memo so the invoice amount and fee are accounted for separately.

Allow a payment-level exception

Before processing, use Remove Processing Fee to waive the fee for this payment. Use Apply Fee to calculate and add it again. Changing the payment method to ACH removes an unprocessed card fee; changing back to card recalculates it.

Process the payment

Use the payment Actions menu to authorize and capture, or use the approved combined action for the demo. Only an approved linked2pay processing transaction can mark the payment paid and close a zero-balance invoice.

Commercial-card data and fallback

When Level 3 is enabled, the integration first sends eligible invoice-line detail as Level 3. A definite gateway rejection with no transaction identifiers falls back to Level 2 and then standard processing. Because the wallet API does not reliably provide card brand, all cards use the same ordered strategy. A timeout or ambiguous result never triggers an automatic fallback or unsafe retry.

Verify both systems

Confirm one approved CC processing transaction in Acumatica and one charge in linked2pay. Repeating the action must not create a second charge.

If Acumatica reports an ambiguous gateway result, verify the linked2pay portal first and use the controlled reconciliation/retry path. Do not simply run the payment again.
