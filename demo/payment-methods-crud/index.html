Payment Methods

Mirror Wallet Changes

Demonstrate create, update, delete, and restore behavior without losing Acumatica payment history.

Add or update a wallet method

Create another card or bank account in linked2pay, or update an existing method while keeping the same Payment Profile ID.

Sync without duplicates

Click Sync with linked2pay. A new Payment Profile ID creates one Acumatica profile; an existing ID updates the matching profile, description, expiration, and active state.

Delete and restore

Delete a method in linked2pay and sync again. Acumatica deactivates it and adds (Deleted) to the description so historical transactions remain valid. If the same Payment Profile ID returns later, Sync reactivates it and removes the deleted marker.

Default and deletion behavior

If the deleted gateway method was the customer default, the integration promotes another active linked2pay method when one is available. Deleting a payment method only in Acumatica does not delete or disable anything in linked2pay.

The Payment Profile ID is the matching key for every card and bank account, including updates, deletion, and restoration.
