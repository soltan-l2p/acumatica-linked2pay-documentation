Recurring Billing

Automate Recurring Invoice Payments

Generate due invoices first, then let linked2pay Auto-Bill charge eligible customers and release their payments.

Demonstrate the second half manually

Open linked2pay Auto-Bill (AR508800), set Invoices Due Before, select the generated invoices, and click Process or Process All.

Show row-level progress

The processing dialog reports each invoice independently. A failed row does not stop the remaining invoices, and a repeated run must not create another charge for an already captured payment.

Configure the unattended daily sequence

  1. Schedule Generate Recurring Transactions first, for example at 6:00 AM.
  2. Schedule linked2pay Auto-Bill after it, for example at 6:30 AM.
  3. Use an automation user with the required AR, payment, release, and processing-center rights.
  4. Test with no active user session and review both schedule histories.

Verify and reconcile

Confirm one linked2pay charge, one approved Acumatica processing transaction, one released payment, and a closed zero-balance invoice. Review Captured But Unreleased and retry release there if capture succeeded but Acumatica release failed.

Auto-Bill is not limited to recurring invoices. Any eligible due AR invoice for an Auto-Bill-enabled customer can be processed.
