VAR Implementation Guide

A repeatable Acumatica VAR framework for discovery, scoping, responsibilities, implementation, testing, go-live, and support transition.

Discovery questionnaire

  • Exact Acumatica version/build/edition/hosting, companies, branches, currencies, customizations?
  • AR card, AR ACH, recurring billing, fees, refunds, voids, or synchronization?
  • Volumes, values, refund/return rates, peaks?
  • Existing customer payment-profile tokens and environments?
  • Cash accounts, posting, approvals, segregation?
  • Owners for network, credentials, accounting, UAT, training, go-live, support?
  • Compliance, retention, change, incident requirements?

Implementation phases

  1. Discover. Scope, build, controls, exclusions.
  2. Design. Centers, methods, cash, approvals, tokens, network, accounting.
  3. Configure sandbox. Complete setup gates.
  4. Test. Execute matrix and resolve exceptions.
  5. Train. AR, administrator, support.
  6. Deploy. Approve change, switch package/credentials, smoke test, reconcile.
  7. Stabilize/handoff. Monitor, document, transfer ownership.

Go-live checklist

  • Package compiled for production URL.
  • Production credentials/webhook key installed securely.
  • No sandbox tokens.
  • Centers/methods/cash/roles/fees approved.
  • Small in-scope rail tests pass.
  • Webhook HTTP 200/status update.
  • Accounting reconciliation signed.
  • Rollback and support bridge active.

Support handoff

Deliver configuration record, package/build IDs, test evidence, known issues, credential/network owners, contacts, escalation, logs, monitoring, and first response. Never include secrets.
