Prerequisites and deployment boundaries

Before You Begin

Confirm version support, modules, credentials, environments, network access, limitations, and implementation effort before publishing the integration.

Supported Acumatica versions

Validated build: Acumatica 2025 R2 build 25.201.0213 on .NET Framework 4.8.

The integration is compiled against that build. Treat other releases, updates, hosting models, or customized payment screens as unvalidated until sandbox regression testing is complete.

  • Rebuild and test before another major release.
  • Revalidate after upgrades or payment-related customization changes.
  • Record exact Acumatica and integration builds.

Required modules, licenses, roles, and permissions

Requirement When needed Minimum expectation
Customization Projects All Import, validate, publish, and unpublish rights.
Cash Management / payment processing All Processing Centers and Payment Methods.
Accounts Receivable Collections Customers, invoices, Payments and Applications, release.
linked2pay merchant account All Enabled customer card and ACH debit services.
Administrator Setup System Administrator or least-privilege equivalent.
AR operator Operations Job actions only; no credential/customization access.

linked2pay account and credentials

  • API login ID and transaction key for the correct merchant/environment.
  • Webhook API key in linked2pay and Acumatica.
  • Required card/ACH services enabled.
  • Tokens created in the same environment as credentials.
Token rule: never reuse contact or payment-profile IDs across environments or merchant accounts.

Sandbox versus production

Environment Base URL Data rule
Sandbox https://test.linked2pay.com Synthetic data and sandbox tokens only.
Production https://api.linked2pay.com Production credentials and tokens.
Build limitation: the base URL is compiled into the integration. Changing credentials does not change environments.

Network, firewall, and webhook requirements

  • Outbound HTTPS TCP 443 from Acumatica to the selected linked2pay URL.
  • Trusted HTTPS access to the Acumatica webhook URL.
  • Preserve X-Api-Key through WAF/proxy/load balancer.
  • Allow the app identity to write App_Data/L2PLogs.

Known compatibility limitations

  • Only build 25.201.0213 is identified as validated.
  • API environment is compile-time.
  • Vendor payments are not available in the current release.
  • Pull customers from linked2pay is unavailable.
  • Generic profile create/update/delete is not implemented.
  • Chargeback and ACH-return accounting documents are not automatic.

Review Known Issues.

Estimated implementation effort

Plan 8–16 implementation hours for a standard single-company deployment: installation, configuration, network coordination, sandbox testing, UAT, and cutover. Add time for multiple entities, custom approvals, upgrades, or conflicts. This is a planning estimate, not an SLA.

Readiness checkpoint

  • Build/modules/roles confirmed.
  • Sandbox credentials and webhook key ready.
  • Network owner assigned.
  • AR card and ACH scope approved.
  • Testing and accounting owners assigned.
