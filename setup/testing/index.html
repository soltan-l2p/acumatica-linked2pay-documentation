User acceptance and production readiness

Testing Matrix

Run positive, negative, security, synchronization, lifecycle, permission, and reconciliation tests before production.

Test execution rules

  • Run in sandbox compiled for https://test.linked2pay.com.
  • Use dedicated test customers, cash accounts, periods, and tokens.
  • Record expected/actual documents, statuses, IDs, batches, and masked evidence.
  • Require signed exceptions for any in-scope test not passed.

Acceptance criteria

  • All in-scope positive/negative tests pass.
  • Accounting approves AR payments, fees, refunds, returns, and reconciliation.
  • Security approves credentials, webhook, roles, logs.
  • Support can locate logs and run first response.
  • Go-live and rollback owners sign.
