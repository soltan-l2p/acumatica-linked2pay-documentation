User acceptance and production readiness

Testing Matrix

Run positive, negative, security, synchronization, lifecycle, permission, and reconciliation tests before production.

Test execution rules

  • Run in sandbox with the processing-center API Base URL set to https://test.linked2pay.com.
  • Use dedicated test customers, cash accounts, periods, invoices, and tokens.
  • Record expected and actual documents, statuses, identifiers, batches, and masked evidence.
  • Repeat critical tests in Modern UI and Classic UI when both are in scope.

Acceptance criteria

  • All in-scope positive, negative, fallback, recovery, and repeat-run tests pass.
  • Accounting approves payments, fees, refunds, returns, defaults, cash accounts, and reconciliation.
  • Security approves credentials, webhook authentication, log level, retention, and roles.
  • Both automation schedules complete unattended in the intended order.
  • Support can locate Trace, webhook inbox/history, file logs, and captured-but-unreleased records.
