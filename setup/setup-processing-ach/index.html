Setup 2D

Create and Test the ACH Processing Center

Create a separate active processing center for ACH debit and credit transactions.

Purpose

Duplicate the approved linked2pay connection pattern for ACH while forcing PAYMETHOD to ACH and card fee to zero.

Prerequisites

  • L2PCC credentials tested.
  • Webhook configured.
  • ACH services enabled for merchant.
  • Approved ACH settlement cash account.

Acumatica navigation path and screen ID

Navigation: Banking > Credit Card Processing > Processing Centers (CA2050PL / CA205000)

Step-by-step instructions

The linked2pay credit card processing center setup is complete. Click Save, then click Test Credentials to confirm that the connection is working. If everything is configured correctly, Acumatica should show a success message.

Finally, create a second processing center for ACH payments. Use the same setup process as the credit card processing center, with these differences:

  • Processing Center ID: L2PACH
  • Name: linked2pay ACH Payments
  • FEE: 0 (ACH payments do not have a processing fee)

When you are done, click Save, then click Test Credentials.

Processing centers are ready. Next, configure the Acumatica payment methods that will use the linked2pay processing centers.

Expected result

The active L2PACH center authenticates successfully, uses ACH payload behavior, and points to the approved ACH cash account.

Validation steps

  • Credential test returns intended client ID.
  • PAYMETHOD remains ACH after refresh.
  • FEE is 0.
  • Webhook key is configured without exposure.
  • Card and ACH centers are separate and active.
