Setup 5

Automation Schedules

Generate recurring invoices first, then let linked2pay Auto-Bill collect every eligible due invoice without an active user session.

Purpose

Use Acumatica's standard scheduling framework. Do not replace the recurring invoice generator: schedule the standard process, then schedule linked2pay Auto-Bill after it.

Recommended order: Generate Recurring Transactions at 6:00 AM, then linked2pay Auto-Bill at 6:30 AM. Adjust the times for the customer's batch window.

Prerequisites

  • Recurring transaction templates and schedules are active.
  • Customers that should be collected have Enable Auto Bill checked.
  • Each customer has an active default L2PCC or L2PACH profile with a cash account and valid tokens.
  • The automation service account can open, process, release, and schedule both screens.

Schedule 1: Generate Recurring Transactions

  1. Open Generate Recurring Transactions (AR504000).
  2. Set the generation filter and confirm the intended schedules appear.
  3. Use the screen's scheduling command to create a daily automation schedule.
  4. Set the run time before Auto-Bill, such as 6:00 AM.
  5. Save, activate, and run one controlled unattended test.

Schedule 2: linked2pay Auto-Bill

  1. Open linked2pay Auto-Bill (AR508800).
  2. Use business date for Invoices Due Before, unless the approved policy specifies another date.
  3. Create a daily automation schedule after invoice generation, such as 6:30 AM.
  4. Save, activate, and test it without an interactive user session.

Auto-Bill processes all eligible due AR invoices for Auto-Bill-enabled customers. The invoice does not have to come from a recurring schedule.

Auto-Bill eligibility

  • Document type is Invoice.
  • Status is Open or Balanced.
  • Due date is on or before the filter date.
  • Customer has Enable Auto Bill checked.
  • Customer default profile is an active linked2pay method with a valid processing center, Contact Profile ID, Payment Profile ID, and cash account.

Monitoring and recovery

  • Review automation schedule history and per-row processing results.
  • Repeated or concurrent runs do not recapture an approved payment.
  • If capture succeeds but release fails, use Captured But Unreleased on AR508800.
  • Correct the release problem, then use Retry Captured Payment Releases. This resumes release without another gateway charge.
  • Use Export Captured But Unreleased for reconciliation evidence.

Validation steps

  • A recurring invoice is generated by the first schedule.
  • The second schedule creates, applies, captures, and releases one payment.
  • The invoice closes when its balance reaches zero.
  • A repeated run creates no duplicate payment or charge.
  • A standard non-recurring due invoice is also eligible when Auto Bill is enabled.
  • A forced release failure appears in recovery and releases after correction without recapture.
