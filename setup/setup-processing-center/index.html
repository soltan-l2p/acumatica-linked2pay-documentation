Setup 2A

Create the Credit Card Processing Center

Create the active linked2pay card processing center and select the integration plugin.

Purpose

Create the Acumatica processing-center record that owns card credentials, payment behavior, fee settings, and the linked2pay plugin.

Prerequisites

  • Integration published/verified.
  • Approved card cash account exists.
  • Card services enabled on the linked2pay merchant.
  • Administrator can maintain Processing Centers.

Acumatica navigation path and screen ID

Navigation: Banking > Credit Card Processing > Processing Centers list, then open the record (CA2050PL / CA205000)

Step-by-step instructions

Open the processing center screen

Now we connect Acumatica to linked2pay.

In Acumatica, go to Banking > Credit Card Processing > Processing Centers (Screen ID: CA2050PL).

This screen stores the gateway connection details. For linked2pay, the processing center becomes the main connection point between Acumatica and your linked2pay account.

Create a new processing center. Click the + button to open the processing center setup form.

You should now see the processing center form (Screen ID: CA205000).

First processing center: credit card payments

Enter the following values into the processing center form:

  • Processing Center ID: L2PCC
  • Name: linked2pay Credit Card Payments
  • Active: Checked
  • Cash Account: Your Acumatica cash account

For Payment Plug-In, click the Search icon and select Linked2PayIntegration.PaymentProcessor.

After you select Linked2PayIntegration.PaymentProcessor, additional fields appear. Keep this Acumatica window open for the next part of the setup.

Field reference

Field or settingRequired valueNotes
Processing Center IDL2PCCRecommended card center ID.
Namelinked2pay Credit Card PaymentsUse clear customer-approved naming.
ActiveCheckedRequired for customer sync and payment processing.
Cash AccountApproved card settlement accountControls reconciliation/accounting.
Payment Plug-InLinked2PayIntegration.PaymentProcessorLoads API, webhook, fee, commercial-card, and Auto-Bill settings.

Expected result

An active L2PCC record opens on CA205000, displays linked2pay settings, and is linked to the approved cash account.

Validation steps

  • Record saves without validation errors.
  • Plugin persists after refresh.
  • Settings include API Username, API Password/Key, API Base URL, Webhook API Key, Payment Method, Processing Fee, Level 2/3, Auto-Bill default, and webhook logging/retention.
  • No credentials appear in screenshots or change tickets.
