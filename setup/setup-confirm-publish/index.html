Setup 1B

Publish and Verify the Integration

Publish the reviewed customization and verify that linked2pay components load in Acumatica.

Purpose

Activate the imported project and prove that the linked2pay payment plugin, fields, and screens are available before configuration begins.

Prerequisites

  • Step 1A import/review complete.
  • Maintenance window and rollback owner approved.
  • Current customization snapshot or package export saved.
  • No users running payment or release processes.

Acumatica navigation path and screen ID

Navigation: More Items > Customization > Customization Projects (SM204505)

Step-by-step instructions

Publish the integration

Return to the Customization Projects screen (Screen ID: SM204505). Select the linked2payintegration project, then click Publish at the top of the page. This publishes the customization project and makes it available in Acumatica.

During publishing, Acumatica may show a warning that says "Active website users will be logged out." This is built-in Acumatica behavior and is safe. Review the warning, then click Publish.

Publishing may take several minutes. Wait for the process to finish. When it is complete, you should see successful log messages like the ones shown here.

The linked2pay customization project is now imported and published. Next, configure the processing center so Acumatica can communicate with linked2pay.

Field reference

Field or settingRequired valueNotes

Payment Plug-InLinked2PayIntegration.PaymentProcessorAvailability proves the assembly loaded.

Expected result

Acumatica reports successful publication and the linked2pay payment plugin appears on Processing Centers.

Validation steps

  • Publish log contains no errors.
  • Processing center plugin selector contains linked2pay.
  • The Customer screen shows the linked2pay Integration tab and its read-only Contact Profile ID.
  • linked2pay Auto-Bill (AR508800) and linked2pay Center (LP102000) open successfully.
  • Modified payment screens open in the enabled Modern and Classic UI modes, as applicable.
  • Existing non-linked2pay payment flows still open normally.
