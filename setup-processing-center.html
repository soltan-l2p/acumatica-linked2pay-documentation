Setup Step 2

Create the Linked2Pay Processing Center

Next, create the processing center that stores your Linked2Pay connection settings in Acumatica.

Now we connect the pieces

Let's set up the processing center.

In Acumatica, go to Banking > Credit Card Processing > Processing Centers (Screen ID: CA2050PL).

This is where Acumatica keeps the connection details for payment gateways. For Linked2Pay, this page becomes the main bridge between Acumatica and your Linked2Pay account.

Start by creating a new processing center. Click the + button to open the processing center setup form.

You should see a form like this. (Screen ID: CA205000).

1st Processing Center - For the Credit Card Payments

Please enter the following information into the processing center form:

  • Processing Center ID: L2PCC
  • Name: Linked2Pay Credit Card Payments
  • Active: Checked
  • Cash Account: Your Acumatica Cash Account

For the Payment Plug-In, click on Search icon and then pick Linked2PayIntegration.PaymentProcessor.

Once you pick Linked2PayIntegration.PaymentProcessor as the Payment Plug-In, you will see the following fields appear. Keep that window open for now.

Linked2Pay Portal

Now we need to go to the Linked2Pay portal and configure the integration settings.

At the bottom of the sidebar, click on API Settings.

or simply use this link: Linked2Pay API Settings Page

You should be able to see Web Service Key under the API Settings section. Copy that Web Service Key.

Acumatica

Now in Acumatica, paste that Web Service Key into the APIKEY field in Acumatica.

Important: This key is the credential that allows Acumatica to communicate with Linked2Pay. If you change the Web Service Key in Linked2Pay, you must update it in Acumatica to maintain the connection.

Important - Recommended

Set the AUTOBILL field to True if you want to enable Auto-Bill for newly created customers.
This acts as the global default value for the Auto-Bill Enabled field on new customer records.

Clearance: You can still manage this settings individually for each customer later. This is just a global default value for newly created customers. So basically, you can override this setting for specific customers as needed. You should set this True if you want to enable Auto-Bill for newly created customers, so that way you don't have to manually enable it for each customer everytime you create one.

FEE field is used to determine how many percent of the credit card processing fee you want to pass on to your customers.
For example, if you set this to 3, then 3% of the credit card processing fee will be added to the invoice total for customers who pay with a credit card. If you set this to 0, then no fee will be added to the invoice total for customers who pay with a credit card.

L2PUSER field will be your Linked2Pay username. This is the username you use to log in to the Linked2Pay portal.

PAYMETHOD field will be Credit Card for the first processing center. This is the Credit Card processing center. You will create a second processing center for ACH payments later.

Now open a new tab in your browser and go to Acumatica. Then go to More Items, under Configuration click on Integration.

In Integration section, under Preferences click on Webhooks. It will bring you to the Webhooks page. (Screen ID: SM304000).

Use Linked2Pay as the webhook name. For the implementation class, click the Search button and select Linked2PayIntegration.Webhooks.L2PWebhookHandler. Then click Save to save the webhook.

Once you save it, you should be able to see URL field. Copy that URL and go back to the Linked2Pay portal.

In Linked2Pay portal, go to API Settings page again and under Webhooks section click on Add Webhook button.

For the Payload URL, paste the URL you copied from Acumatica webhook.

And copy the API Key value (You will paste it in next step in Acumatica).

Make sure you check ACH and Credit Card options. Then click on Save to save the webhook.

Important: If you ever happen to change the API Key in Linked2Pay, you must update it in Acumatica to maintain the webhook connection (for all the Linked2Pay Processing Centers).

Crucial: If you ever change the API Key in Linked2Pay and you don't update it in Acumatica, you won't have payment status updates as they happen with Webhook connections.

For the WEBHOOKKEY, paste the API Key you copied from Linked2Pay Webhook.

We are done with Linked2Pay Credit Card Processing Center setup. Click on Save to save the processing center.

And then click on Test Credentials to make sure the connection is working. You should see a success message if everything is set up correctly.

And finally create another processing center for ACH payments. The steps are the same as above, except for the following changes:

  • Processing Center ID: L2PACH
  • Name: Linked2Pay ACH Payments
  • FEE: 0 (ACH payments do not have a processing fee)

Once you are done, Save and Test Credentials.

Congratulations! You have successfully set up the processing centers.

Now we will move to the last step for setting up payment methods.
