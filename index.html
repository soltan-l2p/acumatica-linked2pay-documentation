Client Presentation

Linked2Pay Integration for Acumatica

Linked2Pay connects Acumatica customers, payment methods, invoices, and payments with the Linked2Pay gateway. The goal is simple: reduce manual payment work, support saved payment profiles, and automate recurring billing.

Customer Boarding Payment Method Sync Auto-Bill Credit Card Fee Webhook Updates
Important: This is an Acumatica customization extension. It does not modify Acumatica source code.

Business Value

What This Solves

Businesses using Acumatica often need to collect payments from customers repeatedly. Linked2Pay helps turn that into a guided, repeatable, and automated workflow.

Less Manual Work

Users do not need to manually copy payment tokens, chase payment status, or process every recurring invoice one by one.

Cleaner Customer Records

Each Acumatica customer can store the Linked2Pay Contact ID and synced payment profile references.

Automated Collections

Auto-Bill can process due invoices for customers that have saved Linked2Pay payment methods.

Simple Architecture

How The Integration Works

Acumatica remains the accounting system of record. Linked2Pay handles gateway-side contact profiles, payment profiles, and transaction processing.

Area Acumatica Linked2Pay
Customer Customer record Contact Profile ID
Payment Method Customer payment method Payment Profile ID for card or ACH
Invoice AR invoice or debit memo Used for payment transaction context
Payment AR payment record Card or ACH transaction
Status Updated by processing and webhook logic Sends transaction result/status
Key idea: Acumatica stores profile IDs and masked payment details only. Full card or bank numbers stay in Linked2Pay.

Setup Guide

Initial Setup

Before using the integration, the Linked2Pay processing center and payment methods must be configured in Acumatica.

1

Create Payment Methods

Create or confirm payment methods such as L2PCC for credit card and L2PACH for ACH.

2

Create Linked2Pay Processing Center

Set up the Linked2Pay processing center and enter the required API settings.

3

Attach Payment Methods

On the processing center, add L2PCC and L2PACH under the payment methods tab and make them active.

4

Test Credentials

Use the Test Credentials action to confirm Acumatica can connect to Linked2Pay successfully.

Customer Workflow

Boarding And Syncing Customers

A customer can be connected to Linked2Pay from the Acumatica Customer screen.

Board To Gateway

Creates a Linked2Pay contact profile from the Acumatica customer. The returned Linked2Pay Contact ID is saved on the customer record.

Sync With Linked2Pay

Reads the customer’s Linked2Pay Contact ID, pulls current contact/payment profile data, and updates Acumatica.

Field Meaning
Linked2Pay Contact ID The customer/contact profile ID from Linked2Pay.
Boarded To Gateway Shows whether the customer has already been created or connected in Linked2Pay.
Enable Auto Bill Controls whether this customer is eligible for automated invoice payments.

Wallet Sync

Payment Method Sync

Linked2Pay payment profiles can be synced into Acumatica as customer payment methods. The Payment Profile ID is the matching key.

Create

If Linked2Pay has a payment profile that does not exist in Acumatica, sync creates it.

Update

If the Payment Profile ID already exists, sync updates the masked card or ACH display details.

Deactivate

If a Linked2Pay profile no longer exists, Acumatica can deactivate the local payment method reference.

Linked2Pay Object Acumatica Result Stored Value
Credit card profile Customer payment method using L2PCC Masked card, expiration date, Payment Profile ID
Bank account profile Customer payment method using L2PACH Masked account, routing number, Payment Profile ID
Note: Deleting a payment method in Acumatica does not delete or disable the payment method in Linked2Pay.

Automation

Auto-Bill Processing

Auto-Bill finds due AR invoices for eligible customers and processes payment through Linked2Pay using the customer’s saved payment method.

1

Invoice Becomes Due

An AR invoice is open or ready to be released and has a due date within the selected processing date.

2

Customer Is Eligible

The customer has Enable Auto Bill turned on and has a default Linked2Pay payment method.

3

Payment Is Created

Acumatica creates an AR payment and applies it to the invoice.

4

Linked2Pay Charges The Customer

The extension sends the transaction to Linked2Pay and records the result back in Acumatica.

Credit card fee: If configured, the extension can apply the credit card processing fee as part of the auto-bill flow.

Status Updates

Webhook Handling

Linked2Pay can send transaction status updates back to Acumatica through a webhook endpoint. This helps Acumatica stay aligned with gateway payment results.

Webhook Requirement Purpose
Endpoint URL The public URL that Linked2Pay calls when payment status changes.
API Key Header Used to validate that the incoming request is allowed.
Transaction Reference Used to find the matching Acumatica payment or Linked2Pay transaction record.
Security: Incoming webhook requests should be validated before Acumatica updates payment status.

Client FAQ

Common Questions

Does this replace Acumatica?

No. Acumatica remains the ERP and accounting system. Linked2Pay handles payment gateway operations.

Are full card numbers stored in Acumatica?

No. Acumatica stores masked display values and Linked2Pay profile IDs.

Can one customer have multiple payment methods?

Yes. A customer can have multiple cards and ACH accounts synced from Linked2Pay.

Does Auto-Bill only pay recurring invoices?

No. Auto-Bill can process eligible due AR invoices, not only invoices created by recurring transactions.

What happens if a payment method is removed in Linked2Pay?

The next sync can update Acumatica so the local payment method reference is no longer active.

Can this run automatically?

Yes. Acumatica processing screens can be scheduled so recurring invoices and Linked2Pay auto-bill can run daily.
