Status Updates

Webhook Handling

Linked2Pay can send transaction status updates back to Acumatica through a webhook endpoint, keeping gateway results aligned with AR payments.

Guided Walkthrough

Explain status updates calmly.

This page is for the handoff moment when someone asks how Acumatica knows what happened after a gateway event. Keep it direct: Linked2Pay calls back, Acumatica validates, then the payment status updates.

The screenshot placeholder can later become the webhook configuration or transaction-status screen.

Webhook Requirement Purpose
Endpoint URL The public URL that Linked2Pay calls when payment status changes.
API Key Header Used to validate that the incoming request is allowed.
Transaction Reference Used to find the matching Acumatica payment or Linked2Pay transaction record.
Security: Incoming webhook requests should be validated before Acumatica updates payment status.
